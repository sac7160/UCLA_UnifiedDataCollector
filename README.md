# UnifiedDataCollector

A system for simultaneously collecting Galaxy Watch8 IMU/audio, a surface contact microphone, and a camera-based fingertip virtual IMU. Built as a multimodal data acquisition pipeline for on-surface handwriting tracking/recognition research (WatchTouch).

## Collected Streams

| Stream                | Source               | Notes                                                |
| --------------------- | -------------------- | ---------------------------------------------------- |
| Watch IMU (acc/gyro)  | Galaxy Watch8, TCP   | Includes a timestamp based on the watch's own clock  |
| Watch audio           | Galaxy Watch8, TCP   | Each frame (40ms) includes its own capture timestamp |
| Surface mic           | USB audio interface  | Surface-contact microphone (ASIN B0F5GGGSNG)         |
| Fingertip virtual IMU | PC camera, MediaPipe | Accel/gyro for all 5 fingertips                      |

    │                                  # run.py + collector/ — kept for reference

`collector/` is a real Python package (each subfolder has an `__init__.py`) —
`run.py` and `collector/` deliberately have different names so that
`import collector.core` never collides with a same-named `.py` file sitting
next to it. See `collector/README.md` and each subpackage's `README.md` for
what's in it.

## Setup

### 1. Build the watch app

Open the `UnifiedDataCollector` project (the `app` module) in Android Studio and **build & install it directly onto the Galaxy Watch**. The watch and the PC must be on the **same WiFi network**.

### 2. PC environment

```bash
pip install -r requirements.txt
```

### 3. Camera calibration (once per camera setup)

```bash
python list_cameras.py               # find your camera's index
python trajectory_calibration.py     # writes calibration.json
```

### 4. Audio interface

Connect the surface mic to the USB audio interface, then check the device/channel:

```bash
python run.py --list-devices
```

## Data Collection Procedure

1. From the `run/` folder, start the collector:
    ```bash
    python run.py
    ```
    Two windows open: an **instructor** window (live waveforms/spectrograms/IMU, REC controls, surface-material presets, label field) and a plain **experimenter** window meant for a second monitor facing the person writing.
2. Launch the watch app, **enter the PC's IP address, and press Start** to begin streaming. (The console prints the PC's IP as `[NET] Watch TCP: <IP>:50005`.)
3. Set the surface material preset and type the current label/stimulus in the instructor window.
4. Press **spacebar once to start recording, press it again to stop** (or use the REC button) — one saved trial spans the full window between the two presses.
5. To quit, close either window or press `Ctrl+C`.

### Key CLI options

| Option                          | Description                                                     |
| ------------------------------- | --------------------------------------------------------------- |
| `--mic-device`, `--mic-channel` | Specify the audio interface device/channel                      |
| `--material`                    | Initial surface-material preset (changeable live in the GUI)    |
| `--dataset-root`                | Trial dataset output path (default: `dataset/`)                 |
| `--trial-margin`                | Margin trimmed inward from both REC start/stop (default: 0.1s)  |
| `--no-camera`                   | Disable the camera (fingertip IMU)                              |
| `--camera-pitch-deg`            | Camera tilt angle (for gravity compensation)                    |
| `--list-devices`                | List audio devices and exit                                     |
| `--verbose`                     | Show per-packet watch IMU arrival logs (chatty; off by default) |

## Output Data Layout

**Session level** (`data/session_YYYYMMDD_HHMMSS_<label>/`) — the full raw recording

```
watch_audio.wav   surface_mic.wav   imu.csv   fingertip_imu.csv   events.csv   sync.json
```

**Trial level** (`dataset/<label>/trial_XXX/`) — one trial per REC start/stop, spanning that
full window (see "Trial boundaries" below)

```
watch_audio.wav   surface_mic.wav   imu.csv   fingertip_imu.csv
```

All trial records accumulate in `dataset/metadata.csv`.

### Trial boundaries

Each saved trial spans the **entire REC window** — from the 1st spacebar press to the 2nd.
Within that window, every individual touch-on/touch-off the surface-mic detector fires is
still logged to that session's `events.csv` (`audio_touch_on` / `audio_touch_off`, alongside
`rec_start` / `rec_end`) with precise, debounce-corrected timestamps — so exact touch timing
is always available even though it no longer trims what gets saved.

### Automatic touch detection

The surface mic drives a band-pass → envelope → threshold+hysteresis → debounce pipeline that
detects finger contact automatically (shown live as the ON/OFF indicator and dB reading in the
instructor window). The noise floor is calibrated once — for `CALIBRATION_DURATION_SEC` after
startup or any material-preset switch, keep the surface untouched while it listens and fixes a
floor value; it does not adapt further until the next recalibration.

## Synchronization

- All streams are aligned to the PC's common reference clock (`time.perf_counter()`).
- Watch IMU/audio also carry the **watch's own clock timestamp** (`watch_ts_ms`), mapped to PC
  time via the RTBGN (session start) and RTEND (session end) reference points — RTEND additionally
  lets a post-session pass correct for watch-clock drift over the session.
- Watch audio frames are timestamped by the watch only once fully buffered, which makes them
  systematically late relative to true capture time; `WATCH_AUDIO_LATENCY_SEC` in
  `collector/core/config.py` corrects for this. It's a fixed, empirically measured constant —
  re-measure it (via `plot_audio.py`) if the watch, USB interface, or OS changes.
- The watch and PC communicate over a **persistent TCP connection** established at session start,
  with automatic reconnection if the connection drops.

## Verification & Diagnostics

- A `[QUALITY]` log is printed at the end of each session. Sessions with an unstable watch connection will show a warning like the one below — **re-collecting is recommended** in that case:
    ```
    [QUALITY] WARNING: watch connection may have stalled during this session ...
    ```
- Use `plot_imu.py` to visually compare a trial's watch IMU against its fingertip virtual IMU for a quick data-quality check:
    ```bash
    python plot_imu.py --imu <trial_dir>/imu.csv --fingertip <trial_dir>/fingertip_imu.csv --finger index
    ```
- Use `plot_audio.py` to check surface_mic/watch_audio alignment for a trial (compare files from the
  same `trial_dir/`, not across a whole session — see "Synchronization" above):
    ```bash
    python plot_audio.py <trial_dir>/surface_mic.wav <trial_dir>/watch_audio.wav
    ```
- Use `index_trajectory_viewer.py` to visualize a recorded fingertip trajectory.
- Use `realtime_viz.py` for a monitoring-only dashboard (no recording) to tune touch-detection
  thresholds/material bands before starting a real session.

---

## Known Limitations

- The fingertip virtual IMU's frame rate is capped around 22-23Hz, limited by the webcam hardware (30fps) and MediaPipe inference speed.
- Surface mic gain depends on the audio interface's hardware preamp — re-check signal level whenever the surface material changes.
- The calibrated noise floor doesn't adapt if ambient noise changes significantly _within_ a session — switch material (even to the same one) to force a recalibration if that happens.
